# 后台模型使用限制配置功能设计

## 概述

在后台管理系统中添加模型使用限制配置功能，允许管理员为每个模型设置访问限制。

## 需求理解

1. **默认免费**：不配置限制时，模型默认可自由使用
2. **多规则支持**：可添加多条限制规则，多组条件需同时满足（AND 关系）
3. **限制类型**：
   - 按请求次数（requests）
   - 按 Token 数（input_tokens）
   - 按金额（cost）
4. **时间周期**（仅 requests 和 input_tokens）：
   - 按天（day）：每天重置
   - 按周（week）：每周重置
   - 按月（month）：每月重置
   - 按小时（hours）：最近 N 小时，需指定小时数
5. **金额限制**：总量限制，无时间周期，花完即失效
6. **价格配置**：仅在选择"按金额"时显示，用于计算费用

## UI/UX 设计

在现有模型表单底部添加"使用限制"区域：

```

│  │ 限制类型: [按请求次数 ▼]          [🗑] │ │
│  │ 时间周期: [按天 ▼]                       │ │
│  │ 限制数值: [____]                         │ │
│  └─────────────────────────────────────────┘ │
│                                                │
│  ┌─────────────────────────────────────────┐ │
```

### 交互设计

1. **默认状态**：只显示"添加限制规则"按钮，页面简洁
2. **添加规则**：点击按钮展开一个新卡片，可配置
3. **编辑规则**：点击卡片内的"编辑"进入编辑模式
4. **删除规则**：点击删除图标确认后移除
5. **按金额特殊处理**：
   - 选择"按金额"时，自动展开价格配置区域
   - 显示输入/输出/缓存 token 单价输入框
   - 不选择"按金额"时，不显示价格配置

### 字段说明

| 字段 | 类型 | 说明 |
|-----|-----|-----|
| 限制类型 | 下拉框 | Free / 按请求次数 / 按 Token 数 / 按金额 |
| 时间周期 | 下拉框 | 按天 / 按周 / 按月 / 按小时（仅限 requests/input_tokens） |
| 限制数值 | 数字输入 | 限制的具体数值 |
| 小时数 | 数字输入 | 当时间周期为"按小时"时需填写 |
| 输入单价 | 数字输入 | 每百万输入 token 的价格（美元） |
| 输出单价 | 数字输入 | 每百万输出 token 的价格（美元） |
| 缓存单价 | 数字输入 | 每百万缓存 token 的价格（美元） |

### 验证规则

1. 限制类型为 Free 时：不显示其他配置
2. 限制类型为"按金额"时：
   - 不显示时间周期选择
   - 必须填写至少输入单价（输出单价可选）
3. 限制类型为"按请求次数"或"按 Token 数"时：
   - 必须选择时间周期
   - 如果选择"按小时"，必须填写小时数
   - 必须填写限制数值（> 0）

## 数据结构

### 后端接口 (config.ts)

现有的 `ModelLimit` 接口已定义：

```typescript
export interface ModelLimit {
  type: 'requests' | 'input_tokens' | 'cost';
  period: 'day' | 'hours' | 'week' | 'month';
  periodValue?: number;  // 当 period='hours' 时，指定小时数
  max: number;           // 最大限制值
}

export interface ProviderConfig {
  // ... 现有字段
  inputPricePer1M?: number;    // 输入 token 每百万价格（美元）
  outputPricePer1M?: number;   // 输出 token 每百万价格（美元）
  cachedPricePer1M?: number;   // 缓存 token 每百万价格（美元）
  limits?: ModelLimit[];       // 使用限制配置
}
```

### 前端状态

使用 React/Hono JSX 状态管理，每个限制规则作为独立对象：

```typescript
interface LimitFormData {
  id: string;              // 唯一标识，用于编辑/删除
  type: 'requests' | 'input_tokens' | 'cost' | 'free';
  period?: 'day' | 'hours' | 'week' | 'month';
  periodValue?: number;
  max?: number;
  // 价格配置（仅 type === 'cost' 时使用）
  inputPricePer1M?: number;
  outputPricePer1M?: number;
  cachedPricePer1M?: number;
}
```

## 实现方案

### 文件修改

1. **src/admin/views/model-form.tsx**：添加限制配置 UI
2. **src/admin/routes/model-form.tsx**：处理表单提交/编辑逻辑

### 实现步骤

1. 在 ModelFormPage 组件中添加状态管理
2. 实现限制规则的增删改 UI
3. 实现按金额时的条件展开
4. 处理表单提交时的数据序列化
5. 后端接收并保存配置

## 边界情况处理

1. **首次添加**：无任何限制时，表单不显示限制区域
2. **编辑已有模型**：加载现有限制配置到表单
3. **删除所有限制**：用户可删除所有规则，恢复为 Free
4. **重复约束**：允许多条相同类型的限制（如多个时间周期不同的请求限制）
5. **空值处理**：未填写的选填字段，保存时剔除

## 成功标准

1. 用户可在后台为模型添加 0 条或多条限制规则
2. 页面默认简洁，只在需要时展开配置
3. 按金额限制时可配置价格
4. 支持单条规则的编辑和删除
5. 配置正确保存到 config.json
